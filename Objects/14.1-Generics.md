## üß≠ 1. What Are Generics?

Generics let you **write code that works with different data types** while maintaining **compile-time type safety**.

Without generics, you would have to use `Object`, which can hold any type but loses type information ‚Äî requiring casting.

---

### üß© Without Generics (Old Way)

```java
import java.util.*;

public class WithoutGenerics {
    public static void main(String[] args) {
        List list = new ArrayList(); // raw type, no type safety
        list.add("Hello");
        list.add(123); // no compile error!

        for (Object obj : list) {
            // Dangerous: might throw ClassCastException
            String str = (String) obj;
            System.out.println(str);
        }
    }
}
```

**Problem:**
When you try to cast `123` (an Integer) to `String`, the program crashes at runtime.
Type errors appear **after** execution begins ‚Äî not safe!

---

### ‚úÖ With Generics (Modern Way)

```java
import java.util.*;

public class WithGenerics {
    public static void main(String[] args) {
        List<String> words = new ArrayList<>(); // type-safe
        words.add("Hello");
        // words.add(123); // ‚ùå compile-time error

        for (String word : words) {
            System.out.println(word.toUpperCase());
        }
    }
}
```

**Benefit:**
The compiler ensures that only `String` objects can go into the `List<String>`.
No casting, no runtime surprises.

## üß© 2. Generic Type Parameters

Generics use **type parameters** in angle brackets (`<>`) to define the placeholder type.

| Symbol | Common Meaning                | Example    |
| ------ | ----------------------------- | ---------- |
| `T`    | Type                          | `Box<T>`   |
| `E`    | Element (used in collections) | `List<E>`  |
| `K`    | Key                           | `Map<K,V>` |
| `V`    | Value                         | `Map<K,V>` |
| `?`    | Unknown type (wildcard)       | `List<?>`  |

---

## üß© 3. Generic Classes

You can create your own classes that accept a type parameter.

### Example: `Box<T>`

```java
public class Box<T> {
    private T content;

    public void set(T content) {
        this.content = content;
    }

    public T get() {
        return content;
    }
}
```

### Using the Generic Class

```java
public class TestBox {
    public static void main(String[] args) {
        Box<String> stringBox = new Box<>();
        stringBox.set("Hello, Generics!");
        System.out.println(stringBox.get());

        Box<Integer> intBox = new Box<>();
        intBox.set(42);
        System.out.println(intBox.get());
    }
}
```

**Output:**

```
Hello, Generics!
42
```

‚úÖ You can reuse the same class for any type, and the compiler ensures type correctness.

## üß© 4. Generic Methods

You can also make **individual methods** generic, even inside non-generic classes.

### Example: A Utility Class with a Generic Method

```java
public class GenericPrinter {
    public static <T> void printArray(T[] array) {
        for (T item : array) {
            System.out.print(item + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        String[] names = {"Alice", "Bob", "Charlie"};
        Integer[] numbers = {1, 2, 3, 4, 5};

        printArray(names);
        printArray(numbers);
    }
}
```

**Output:**

```
Alice Bob Charlie 
1 2 3 4 5 
```

‚úÖ `<T>` before the return type means "this method is generic and T is a type parameter."

## üß© 5. Generic Interfaces

Interfaces can also use generics to define flexible contracts.

### Example: `Container<T>` Interface

```java
interface Container<T> {
    void add(T item);
    T get();
}
```

### Implementation Example

```java
class SimpleContainer<T> implements Container<T> {
    private T item;

    public void add(T item) {
        this.item = item;
    }

    public T get() {
        return item;
    }
}

public class InterfaceExample {
    public static void main(String[] args) {
        Container<String> stringContainer = new SimpleContainer<>();
        stringContainer.add("Data");
        System.out.println(stringContainer.get());
    }
}
```

**Output:**

```
Data
```

## üß© 6. Bounded Type Parameters

Sometimes you want to **restrict** the type parameter to a certain family of classes.

### Example: Using `extends` (Upper Bound)

```java
class MathUtil {
    public static <T extends Number> double add(T a, T b) {
        return a.doubleValue() + b.doubleValue();
    }
}

public class BoundedExample {
    public static void main(String[] args) {
        System.out.println(MathUtil.add(10, 20));        // int
        System.out.println(MathUtil.add(5.5, 4.5));      // double
        // MathUtil.add("Hi", "There"); // ‚ùå compile error
    }
}
```

‚úÖ Only `Number` or its subclasses (`Integer`, `Double`, etc.) are allowed.

---

### Example: Using `super` (Lower Bound)

```java
import java.util.*;

public class LowerBoundExample {
    public static void addNumbers(List<? super Integer> list) {
        list.add(1);
        list.add(2);
        list.add(3);
    }

    public static void main(String[] args) {
        List<Number> numbers = new ArrayList<>();
        addNumbers(numbers);
        System.out.println(numbers);
    }
}
```

‚úÖ `? super Integer` means ‚Äúa list that can accept `Integer` or any of its supertypes (like `Number` or `Object`).‚Äù

## üß© 7. Wildcards (`?`)

The **wildcard** `?` represents an **unknown type**.
It‚Äôs useful for reading data when you don‚Äôt need to know the exact type.

### Example: Unbounded Wildcard

```java
import java.util.*;

public class WildcardExample {
    public static void printList(List<?> list) {
        for (Object item : list) {
            System.out.println(item);
        }
    }

    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
        List<Integer> numbers = Arrays.asList(10, 20, 30);

        printList(names);
        printList(numbers);
    }
}
```

‚úÖ `List<?>` can accept a list of any type, but you can‚Äôt add new elements to it (other than `null`).

## üß† 8. Why Use Generics?

| Advantage              | Description                                    |
| ---------------------- | ---------------------------------------------- |
| **Type Safety**        | Catches invalid types at compile time.         |
| **Eliminates Casting** | No need for `(Type)` conversions.              |
| **Code Reuse**         | One class or method works for many data types. |
| **Cleaner Code**       | More readable, less boilerplate.               |

## üß© 9. Real-World Example: Generic Pair Class

```java
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }

    public String toString() {
        return key + " = " + value;
    }
}

public class PairExample {
    public static void main(String[] args) {
        Pair<String, Integer> age = new Pair<>("Alice", 30);
        Pair<Integer, String> code = new Pair<>(404, "Not Found");

        System.out.println(age);
        System.out.println(code);
    }
}
```

**Output:**

```
Alice = 30
404 = Not Found
```

‚úÖ `Pair<K,V>` is a generic class with **two type parameters** ‚Äî just like `Map<K,V>` in Java‚Äôs Collections Framework.
